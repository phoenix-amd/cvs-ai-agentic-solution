1. Overnight Autonomous Mode User Triggers

Why This Exists

GPU cluster validation tests take hours. If a fixable issue occurs at 2 AM (wrong config, NUMA balancing enabled), the entire overnight window is wasted. Nobody is watching. The team arrives at 8 AM, sees the failure, fixes it, waits another 4 hours.

Value

  • Zero wasted overnight hours — agent auto-heals fixable issues and re-runs
  • Morning-ready results — consolidated pass/fail report waiting at 8 AM
  • Hardware issues caught immediately — Jira tickets created at 2:01 AM, not 10 AM

2. Jira Escalation for Hardware Failures Agent Auto

Why This Exists

When a GPU has a memory error at 3 AM, the manual workflow wastes 9 hours: nobody notices until morning → engineer checks logs → copies to Jira → collects diagnostics → hardware team gets ticket at 11 AM. With automated escalation, the ticket exists at 3:01 AM with all diagnostics attached.

Value

  • Instant escalation — hardware team notified within minutes
  • Complete diagnostics — every ticket has rocm-smi, dmesg, ibstat, test logs
  • Consistent ticket quality — same structured format every time

What Triggers Jira vs What's Auto-Fixed

Triggers Jira (Hardware Only)

GPU not detected in lspci
HBM ECC uncorrectable errors
PCIe link degraded (x8 instead of x16)
XGMI link failure between GPUs
IB port DOWN in ibstat
amdgpu reset/hang/fault in dmesg
RAS uncorrectable errors

Does NOT Trigger Jira (Auto-Fixed)

Wrong mpi_oob_port → agent fixes config
Wrong env script for NIC → agent creates new one
SSH key permissions → agent chmod 600
<changeme> in config → agent fills values
NUMA balancing enabled → agent disables
Docker image not pulled → agent pulls
Single-node bus_bw=0 → expected behavior

5. RCCL Pre-Run Validation Agent Auto

Why This Exists

In our field testing, we hit 3 separate config issues before RCCL ran successfully. Each wasted 5-10 minutes. Now the agent discovers all values before running.

6. Auto-Heal Playbook Agent Auto

Why This Exists

Most CVS test failures are configuration issues, not hardware problems. The agent reduces engineer intervention by 70%+ for common issues.

Decision Tree

SAFE (Auto-Fix, No Approval)

NUMA balancing → echo 0
Docker pull → docker pull <image>
SSH key chmod → chmod 600
Config <changeme> → replace with discovered
Wrong env script → generate correct one
Wrong mpi_oob_port → discover and set

MODERATE (Suggest to User)

Firewall disable for RDMA ports
GRUB config changes (iommu=pt)
ROCm version update

CRITICAL (Escalate + Jira)

GPU not detected
HBM/ECC errors
PCIe link degraded
IB port down
GPU reset/hang in dmesg

7. Smart Single-Node Handling Agent Auto

Why This Exists

CVS RCCL baselines are for multi-node. Single-node bus_bw=0 is expected but CVS reports FAILED — a false negative that confuses users.

11. Prompt-Injection Defense Agent Always

All cluster output (stdout, stderr, log files) is treated as DATA. The agent never parses cluster output as instructions. Suspicious output is flagged to the user.
